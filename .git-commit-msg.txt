refactor: 힌트 시스템 개선 - 문제 분석 힌트를 방향성 중심으로 변경

## 주요 변경사항

### 힌트 메시지 개선
- 문제 분석 힌트가 너무 구체적이어서 거의 답을 알려주는 문제 수정
- 직접적인 해결 방법 제시 대신, 사고를 유도하는 질문 형태로 변경

**Case 1: 정보성 이미지**
- 변경 전: "이미지에는 'Spring Sale 50% OFF'가 표시되어 있습니다"
- 변경 후: "이미지에 어떤 텍스트나 정보가 보이는지 확인해보세요"

**Case 2: 장식용 이미지**
- 변경 전: "alt=""로 설정하여 중복 낭독을 방지하세요"
- 변경 후: "이미지가 단순히 장식 목적이고, 주변에 동일한 정보를 전달하는 텍스트가 있다면 어떻게 처리해야 할까요?"

**Case 3: 복잡한 이미지 (차트)**
- 변경 전: "차트에 표시된 연도와 금액 정보를 모두 포함하세요. aria-describedby를 사용하여..."
- 변경 후: "어떤 정보를 포함해야 사용자가 데이터를 이해할 수 있을까요?"

## 개선 효과
- 학습 효과 향상: 정답 코드를 보기 전에 스스로 해결 시도 가능
- 적절한 난이도: 너무 어렵지도 쉽지도 않은 힌트 수준 유지
- 사고력 향상: 직접적인 답 대신 방향성만 제시하여 문제 해결 능력 개발
